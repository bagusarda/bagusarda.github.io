Industrial 3-Phase Power Monitoring IoT Panel (BLKPP DIY)

Proyek integrasi dan perakitan panel pemantauan kelistrikan 3 fasa skala industri untuk fasilitas BLKPP DIY. Sistem ini dirancang untuk membaca penggunaan daya, tegangan, dan arus secara akurat dari panel utama ruangan yang sudah ada, kemudian mendistribusikan data tersebut secara lokal maupun remote (IoT).

Fitur Sistem:

  • Pengukuran parameter 3 fasa (V, I, P) secara komprehensif.
  • Monitoring lokal *real-time* via LCD Multiple Electric Meter.
  • Transmisi data telemetri ke custom dashboard (iot.sibukin.com).
  • Integrasi *non-intrusive* menggunakan Current Transformer (CT) bawaan ruangan.

Komponen & Tech Stack:

  • Data Communication: Modbus USR-DR134 (RS485 to Ethernet)
  • Measurement: LCD Multiple Electric Meter & Current Transformer (CT)
  • Power Supply: PSU HD-15-12 (12V)
  • Platform: Custom Dashboard (iot.sibukin.com)
  • Hardware: Terminal block, Casing box, Kabel & Skun

Implementasi Jaringan & Komunikasi Data:

Keberhasilan sistem ini bergantung pada akuisisi data yang presisi. Arus AC tinggi dari listrik 3 fasa diturunkan secara proporsional oleh Current Transformer (CT) yang terpasang di panel ruangan agar dapat dibaca dengan aman oleh LCD Multiple Electric Meter. Proses wiring menggunakan terminal block dan skun kabel memastikan koneksi standar industri yang aman dan minim noise.

Untuk menjembatani instrumen lokal dengan ekosistem IoT, Modbus USR-DR134 diimplementasikan sebagai gateway. Perangkat ini mengonversi protokol Modbus RTU/RS485 dari meteran listrik menjadi paket data TCP/IP. Data kelistrikan ini kemudian dipublikasikan ke server iot.sibukin.com, memungkinkan staf BLKPP DIY memantau fluktuasi beban listrik 3 fasa di 3 ruangan berbeda secara terpusat.

Smart Home IoT: Voice Control & Energy Monitoring

Sistem otomatisasi rumah pintar berbasis Internet of Things (IoT) yang memungkinkan kontrol lampu via Google Voice Assistant. Selain kontrol suara, sistem ini dilengkapi fitur monitoring daya listrik real-time untuk memantau penggunaan arus, tegangan, dan daya (Watt) secara akurat menggunakan kombinasi sensor ZMPT101B dan ACS712.

Fitur Utama:

  • Kontrol lampu via perintah suara (IFTTT & Google Assistant)
  • Monitoring parameter listrik (V, I, P) di aplikasi mobile Blynk
  • Tampilan data lokal *real-time* via LCD 16x2
  • Kontrol manual *override* via aplikasi Blynk

Tech Stack:

  • Microcontroller: Wemos D1 Mini (ESP8266)
  • Sensors: ZMPT101B (Voltage) & ACS712 (Current)
  • Platforms: Blynk IoT, IFTTT, Google Home
  • Languages: C++ (Arduino IDE)

Alur Kerja & Hasil Analisis:

Sistem otomatisasi ini menggunakan platform IFTTT sebagai perantara web request (Webhooks API) ketika menerima trigger perintah suara dari Google Assistant, yang kemudian diteruskan langsung ke server Blynk Cloud untuk mengubah state relay pada pin D5 Wemos D1 Mini.

Untuk kalkulasi daya, data analog dari sensor arus (ACS712) dan sensor tegangan (ZMPT101B) diproses langsung oleh Wemos D1 Mini. Data hasil konversi tersebut kemudian dikalibrasi secara internal melalui kode pemrograman sebelum dikirimkan secara real-time ke dashboard Blynk IoT.

Soil Moisture Monitoring: Blynk IoT & Local Web Server

Sistem monitoring kelembapan tanah berbasis Internet of Things (IoT) yang dirancang untuk mendeteksi kadar air tanah secara real-time. Sistem ini mengintegrasikan mikrokontroler dengan sensor tanah untuk mendukung optimalisasi perawatan tanaman, di mana data hasil pengukuran dapat dipantau secara offline melalui LCD fisik maupun secara online melalui aplikasi Blynk dan halaman local web server.

Fitur Utama:

  • Monitoring nilai ADC, Tegangan, dan Status tanah secara real-time
  • Tampilan persentase kelembapan (0-100%) pada halaman web lokal
  • Antarmuka lokal menggunakan LCD 16x2 dengan modul I2C
  • Integrasi dual-sistem online via Blynk Cloud dan HTTP Web Server

Tech Stack:

  • Microcontroller: Wemos D1 Mini / NodeMCU (ESP8266)
  • Sensors: Soil Moisture Sensor (Analog Output A0)
  • Display: LCD 16x2 dengan Modul I2C
  • Platforms: Blynk IoT & HTTP Web Server Lokal
  • Languages: C++ (Arduino IDE)

Alur Kerja & Hasil Analisis:

Sistem bekerja dengan membaca sinyal analog dari sensor kelembapan tanah melalui pin A0 pada ESP8266 dengan range nilai ADC antara 0 hingga 1023. Data digital hasil konversi tersebut diproses langsung secara internal oleh mikrokontroler untuk dihitung ke dalam satuan tegangan nyata (0-5 Volt) serta skala persentase kelembapan tanah.

Kondisi tanah kemudian dikategorikan berdasarkan nilai pembacaan, di mana nilai ADC lebih besar atau sama dengan 510 akan menetapkan status Kering, sedangkan nilai kurang dari 510 memicu status Basah. Data status dan parameter numerik ini kemudian dikirim secara berkala ke server Blynk menggunakan fungsi `Blynk.virtualWrite()` serta dirender secara lokal ke halaman web server berbasis IP Address yang dapat diakses via browser.

Custom Smartplug Power Monitoring & Actuation (OpenBeken)

Proyek ini berfokus pada modifikasi (reverse engineering) perangkat smartplug komersial agar dapat dipantau dan dikendalikan menggunakan ekosistem IoT kustom. Melalui proses flashing firmware ke OpenBeken (OBK), data kelistrikan vital seperti tegangan, daya, dan arus divisualisasikan secara real-time melalui Klenik Dashboard (klenik.ichibot.id).

Fitur Sistem:

  • Real-time telemetry kelistrikan (Tegangan, Arus, Daya).
  • Remote control aktuasi (On/Off) via dashboard.
  • Log aktivitas state smartplug secara real-time.
  • Custom firmware tanpa limitasi cloud bawaan pabrik.

Tech Stack & Tools:

  • Platform: Klenik Dashboard (klenik.ichibot.id)
  • Firmware: OpenBeken (OBK)
  • Protocol & Broker: MQTT (mqtt.ichibot.id)
  • Hardware: Smartplug Komersial (ESP/BK7231 based)
  • Flashing Method: Komunikasi Serial UART (TX/RX)

Implementasi & Analisis Modifikasi:

Tantangan utama dalam proyek ini berada pada tahap modifikasi hardware. Smartplug harus dibongkar secara fisik untuk mengakses pinout UART (TX, RX, VCC, GND) guna menimpa firmware bawaan pabrik dengan OpenBeken menggunakan USB to TTL adapter. Penggunaan OpenBeken memungkinkan perangkat keras murah berbasis chipset seperti BK7231 untuk lepas dari ketergantungan cloud aslinya dan terintegrasi penuh ke server lokal atau kustom.

Setelah proses flashing berhasil, tahapan krusial selanjutnya adalah melakukan konfigurasi MQTT Topic pada firmware agar perangkat dapat berkomunikasi lancar dengan server mqtt.ichibot.id. Data payload yang dikirimkan kemudian di-*parsing* oleh Klenik Dashboard untuk ditampilkan dalam bentuk gauge metrik dan antarmuka switch, menghasilkan sistem kendali kelistrikan berlatensi rendah yang sepenuhnya independen.

Multi-Platform IoT Actuation: Web Server, Blynk, & Adafruit IO (MQTT)

Proyek eksperimental ini mengimplementasikan kendali aktuasi 4 buah LED menggunakan mikrokontroler Wemos D1 Mini (ESP8266) yang diuji melalui tiga arsitektur dan platform IoT berbeda. Fokus proyek ini adalah melakukan komparasi performa kontrol aktuasi antara sistem local web server, platform cloud ecosystem (Blynk), serta arsitektur berbasis broker MQTT (Adafruit IO).

Arsitektur Pengujian:

  • Local Web Server: Wemos D1 Mini bertindak sebagai HTTP server lokal.
  • Blynk Cloud: Integrasi API platform via Virtual Pin (V1-V4) secara online.
  • Adafruit IO: Implementasi protokol MQTT via arsitektur publish/subscribe pada feeds.

Tech Stack:

  • Microcontroller: Wemos D1 Mini (ESP8266)
  • Actuators: 4x LED & 4x Resistor 220 Ohm
  • Protocols: HTTP Rest API & MQTT (Port 1883)
  • Platforms: Blynk IoT, Adafruit IO Core
  • Languages: C++ (Arduino IDE)

Alur Kerja & Hasil Analisis Komparasi:

Pada pengujian Local Web Server, kendali dilakukan dengan memproses string request pada header HTTP ketika tombol diclick. Sistem ini andal tanpa ketergantungan internet luar, namun terbatas pada cakupan area Wi-Fi lokal. Sementara pada pengujian Blynk IoT, transisi data memanfaatkan pustaka Blynk untuk sinkronisasi state pin digital secara real-time dan interaktif via aplikasi mobile.

Untuk efisiensi data tertinggi, protokol MQTT via Adafruit IO terbukti paling optimal. Wemos cukup melakukan subscribe ke satu feed terpusat (/feeds/LED). Ketika toggle switch pada dashboard Adafruit diubah, broker langsung mendorong data payload pendek ("ON" / "OFF"), menghasilkan latensi waktu respon aktual yang sangat minim dibandingkan metode HTTP request biasa.

Multi-Platform Climate Monitoring: Local Webpage, Blynk, & ThingSpeak

Proyek ini mengintegrasikan mikrokontroler Wemos D1 Mini (ESP8266) dengan sensor DHT11 untuk memantau parameter suhu udara dan kelembapan lingkungan secara real-time. Melalui eksperimen ini, dilakukan pengujian distribusi data menggunakan tiga metode interwajah terpisah, yaitu web server lokal statis, widget interaktif pada cloud Blynk IoT, serta visualisasi data analitik berbasis grafik tren waktu pada platform ThingSpeak.

Arsitektur Integrasi:

  • Local Webpage: Menyediakan server HTTP internal statis dengan auto-refresh halaman setiap 10 detik.
  • Blynk IoT Dashboard: Visualisasi data real-time menggunakan widget Gauge terhubung via Virtual Pin V5 dan V6.
  • ThingSpeak Cloud: Penyimpanan data berbasis channel virtual untuk tracking grafik analitik berkala per 30 detik.

Tech Stack:

  • Microcontroller: Wemos D1 Mini (ESP8266)
  • Sensors: DHT11 (Temperature & Humidity)
  • Protocols: HTTP REST API & ThingSpeak Channel API
  • Platforms: Blynk IoT Cloud, ThingSpeak Analytics
  • Languages: C++ (Arduino IDE)

Alur Kerja & Hasil Analisis Sistem:

Pemrosesan data dimulai dari pembacaan berkala parameter fisik suhu dan kelembapan oleh sensor DHT11 melalui pin digital D5. Data mentah tersebut divalidasi secara internal sebelum dipetakan ke masing-masing arsitektur komunikasi. Pada antarmuka Webpage Lokal, data langsung dirender ke dalam kerangka dokumen HTML statis yang dikombinasikan dengan aset vektor SVG untuk visualisasi data tanpa koneksi internet luar.

Untuk akses jarak jauh berskala global, integrasi Blynk IoT and ThingSpeak memanfaatkan modul Wi-Fi core ESP8266 untuk melakukan transmisi payload data numerik secara berkala. Dashboard Blynk menyediakan respon pembacaan data yang dinamis via widget mobile, sedangkan server cloud ThingSpeak berfungsi optimal dalam merekam data log jangka panjang ke dalam grafik field analitik, mempermudah pengguna memantau fluktuasi iklim mikro secara berkala.
